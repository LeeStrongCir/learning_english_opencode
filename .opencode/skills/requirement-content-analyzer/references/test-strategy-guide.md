# 测试策略指南

本文件是 `requirement-content-analyzer` 的测试策略参考，在推荐测试类型和自动化方案时参考。

## 1. 测试类型详解

### 1.1 功能测试
**适用场景**：所有需求的基础

**测试内容**：
- 正常流程验证（Happy Path）
- 异常流程验证（Exception Path）
- 边界值验证（Boundary Value）
- 等价类划分（Equivalence Partitioning）
- 状态转换验证（State Transition）

**常用方法**：
- 等价类划分法
- 边界值分析法
- 判定表法
- 状态迁移图
- 场景法

### 1.2 接口测试
**适用场景**：前后端分离、微服务架构、API 产品

**测试内容**：
- 接口功能验证（请求/响应正确性）
- 参数校验（必填、格式、范围、类型）
- 异常处理（参数缺失、格式错误、超时）
- 权限验证（未授权、越权访问）
- 幂等性验证（重复请求、重试机制）
- 限流降级（高频请求、服务降级）

**常用工具**：
- Postman / Newman
- pytest + requests
- JMeter（接口 + 性能）
- REST Assured

### 1.3 性能测试
**适用场景**：有性能指标要求、高并发场景、大数据量处理

**测试内容**：
- 响应时间（P50/P90/P95/P99）
- 吞吐量（TPS/QPS）
- 并发用户数
- 资源利用率（CPU、内存、磁盘、网络）
- 稳定性（长时间运行是否内存泄漏）
- 容量规划（最大支持多少数据/用户）

**常用工具**：
- JMeter
- Locust
- k6
- Gatling

### 1.4 安全测试
**适用场景**：涉及用户数据、支付、权限控制、对外暴露的 API

**测试内容**：
- 认证安全（密码强度、暴力破解防护、Token 安全）
- 授权安全（水平越权、垂直越权）
- 数据安全（传输加密、存储加密、敏感信息脱敏）
- 输入安全（SQL 注入、XSS、CSRF、文件上传）
- 会话安全（会话固定、会话超时、并发登录）
- 日志安全（敏感信息不泄露到日志）

**常用工具**：
- OWASP ZAP
- Burp Suite
- SQLMap
- Nmap

### 1.5 兼容性测试
**适用场景**：Web 应用、移动端应用、多平台支持

**测试内容**：
- 浏览器兼容（Chrome、Firefox、Safari、Edge）
- 操作系统兼容（Windows、macOS、Linux、iOS、Android）
- 设备兼容（手机、平板、PC、不同分辨率）
- 版本兼容（新旧版本共存、向后兼容）
- 网络兼容（4G/5G/WiFi/弱网）

**常用工具**：
- BrowserStack
- Sauce Labs
- Selenium Grid
- 云测平台

### 1.6 异常测试
**适用场景**：所有系统，特别是分布式系统

**测试内容**：
- 网络异常（断网、超时、弱网、DNS 异常）
- 服务异常（依赖服务不可用、数据库异常、缓存异常）
- 数据异常（脏数据、空数据、格式错误、超长数据）
- 并发异常（竞态条件、死锁、数据不一致）
- 降级恢复（服务降级后的恢复、数据补偿）

**常用方法**：
- 混沌工程（Chaos Engineering）
- 故障注入
- Mock 服务异常

## 2. 自动化可行性评估指南

### 2.1 评估维度

| 维度 | 高可行性 | 中可行性 | 低可行性 |
|------|----------|----------|----------|
| UI 稳定性 | UI 稳定，变更少 | UI 偶尔变更 | UI 频繁变更 |
| 接口清晰度 | API 文档完善，接口稳定 | 有部分文档 | 无文档，接口频繁变 |
| 数据准备 | 测试数据易准备，有测试环境 | 需要准备但可行 | 数据准备复杂，依赖外部 |
| 执行频率 | 每次回归都执行 | 每周/每版本执行 | 偶尔执行 |
| 验证方式 | 结果明确可验证 | 需要部分人工判断 | 主要依赖人工判断 |

### 2.2 推荐策略

| 层级 | 自动化优先级 | 推荐工具 |
|------|-------------|----------|
| 单元测试 | 最高 | pytest, JUnit, Jest |
| 接口测试 | 高 | pytest + requests, Postman |
| UI 测试 | 中 | Playwright, Selenium, Cypress |
| 性能测试 | 中 | JMeter, Locust, k6 |
| 安全测试 | 低 | OWASP ZAP, Burp Suite |
| 兼容性测试 | 低 | BrowserStack, 云测平台 |

### 2.3 自动化 ROI 计算

```
自动化收益 = 手动执行时间 × 执行频率 - 自动化开发时间 - 自动化维护时间
```

- 收益 > 0：建议自动化
- 收益 < 0：不建议自动化

## 3. 测试策略选择矩阵

| 需求特征 | 推荐测试类型 | 优先级 |
|----------|-------------|--------|
| 核心业务流程 | 功能测试 + 接口测试 | P0 |
| 用户数据/支付 | 功能测试 + 安全测试 | P0 |
| 高并发场景 | 功能测试 + 性能测试 | P0 |
| 多端支持 | 功能测试 + 兼容性测试 | P1 |
| 外部依赖 | 功能测试 + 异常测试 | P1 |
| 数据迁移 | 功能测试 + 数据验证 | P1 |
| UI 优化 | 功能测试 + 兼容性测试 | P2 |
| 文案调整 | 功能测试（抽查） | P2 |

## 4. 场景设计方法

### 4.1 正常流程（Happy Path）
- 用户按预期操作的完整路径
- 所有输入都是合法的
- 所有依赖服务都正常

### 4.2 异常流程（Exception Path）
- 用户输入错误
- 依赖服务不可用
- 网络异常
- 权限不足
- 数据不存在

### 4.3 边界场景（Boundary）
- 最大值、最小值
- 空值、零值
- 临界时间点
- 临界数量
- 格式边界

### 4.4 并发场景（Concurrent）
- 同一账号多地登录
- 同时操作同一数据
- 高并发请求
- 定时任务与用户操作冲突

## 5. 测试工作量估算参考

| 测试类型 | 工作量占比 | 说明 |
|----------|-----------|------|
| 功能测试 | 40-60% | 基础，必做 |
| 接口测试 | 10-20% | 前后端分离项目必做 |
| 性能测试 | 5-15% | 有性能指标要求时做 |
| 安全测试 | 5-10% | 涉及用户数据/支付时做 |
| 兼容性测试 | 5-10% | 多端支持时做 |
| 异常测试 | 5-10% | 分布式系统必做 |
| 回归测试 | 10-20% | 每次发版必做 |

注：具体占比根据项目特点调整。
