# 测试设计方法参考

本文档详细介绍各种黑盒测试技术，供测试用例生成时参考。

## 1. 等价类划分 (Equivalence Partitioning)

### 原理
将输入域划分为若干等价类，从每个类中选取代表性值进行测试。

### 步骤
1. 识别输入条件
2. 确定有效等价类（符合需求的输入）
3. 确定无效等价类（不符合需求的输入）
4. 为每个等价类设计测试用例

### 示例：年龄输入 (18-60)
| 等价类 | 类型 | 代表值 |
|--------|------|--------|
| 18-60 | 有效 | 25, 40 |
| <18 | 无效 | 17, 0, -1 |
| >60 | 无效 | 61, 100 |
| 非数字 | 无效 | abc, @#$ |

---

## 2. 边界值分析 (Boundary Value Analysis)

### 原理
错误往往发生在输入范围的边界上，需要重点测试边界及其附近的值。

### 测试点
对于范围 [min, max]：
- 正好等于 min
- 正好等于 max
- 略小于 min (min-1)
- 略大于 max (max+1)
- 范围内的典型值

### 示例：密码长度 (6-20位)
| 测试值 | 说明 | 预期 |
|--------|------|------|
| 5 | 小于最小值 | 拒绝 |
| 6 | 最小值 | 接受 |
| 7 | 最小值+1 | 接受 |
| 19 | 最大值-1 | 接受 |
| 20 | 最大值 | 接受 |
| 21 | 大于最大值 | 拒绝 |

---

## 3. 决策表测试 (Decision Table Testing)

### 适用场景
当业务逻辑涉及多个条件的组合判断时。

### 构建步骤
1. 识别所有条件桩（输入条件）
2. 识别所有动作桩（系统行为）
3. 列出所有条件组合
4. 为每个组合确定预期动作
5. 简化不可能或不合理的组合

### 示例：用户登录权限
| 条件 | 规则1 | 规则2 | 规则3 | 规则4 |
|------|-------|-------|-------|-------|
| 用户名正确 | Y | Y | N | N |
| 密码正确 | Y | N | Y | N |
| 账号已激活 | Y | Y | Y | N |
| **动作** | | | | |
| 登录成功 | ✓ | ✗ | ✗ | ✗ |
| 提示密码错误 | ✗ | ✓ | ✗ | ✗ |
| 提示用户不存在 | ✗ | ✗ | ✓ | ✗ |
| 提示账号未激活 | ✗ | ✗ | ✗ | ✓ |

## 4. 状态转换测试 (State Transition Testing)

### 适用场景
系统存在明确的状态机，如订单状态、审批流程等。

### 测试要点
1. 覆盖所有合法状态转换
2. 测试非法状态转换
3. 测试状态循环
4. 测试终止状态

### 示例：订单状态
```
待支付 → 已支付 → 配送中 → 已签收
   ↓         ↓
已取消    退款中 → 已退款
```

**测试用例**：
- 正常流程：待支付→已支付→配送中→已签收
- 取消流程：待支付→已取消
- 退款流程：已支付→退款中→已退款
- 非法转换：已签收→待支付（应拒绝）

---

## 5. 用户旅程测试 (User Journey Testing)

### 原理
从真实用户视角出发，测试端到端的完整业务流程。

### 设计方法
1. 识别用户角色（Persona）
2. 定义用户目标
3. 绘制用户操作路径
4. 识别关键接触点
5. 考虑替代路径和异常路径

### 示例：电商购物流程
```
浏览商品 → 加入购物车 → 登录/注册 → 填写地址 → 选择支付 → 支付成功 → 查看订单
```

**变体**：
- 游客购买 vs 会员购买
- 使用优惠券 vs 不使用
- 单笔订单 vs 合并支付

---

## 6. 异常场景测试 (Exception Testing)

### 常见异常类型

#### 6.1 数据异常
- 空值/NULL
- 超长输入
- 特殊字符（SQL注入、XSS）
- 格式错误（错误日期、错误邮箱）
- 重复提交

#### 6.2 系统异常
- 网络断开/超时
- 服务不可用（503）
- 数据库连接失败
- 磁盘空间不足
- 内存溢出

#### 6.3 权限异常
- 未登录访问受限页面
- 普通用户访问管理员功能
- 跨租户数据访问
- Token 过期/失效

#### 6.4 并发异常
- 同一账号多地登录
- 库存超卖（并发下单）
- 重复领取优惠券

---

## 7. 探索式测试启发 (Exploratory Testing Heuristics)

### 常用启发式方法

#### CRUSSPIC 质量模型
- **C**apability: 系统能做什么
- **R**eliability: 系统是否稳定
- **U**sability: 系统是否易用
- **S**ecurity: 系统是否安全
- **S**calability: 系统能否扩展
- **P**erformance: 系统性能如何
- **I**nstallability: 安装部署是否顺利
- **C**ompatibility: 与其他系统兼容性

#### 测试技巧
- **反转测试**：把正常操作反过来做
- **随机测试**：随机输入、随机点击
- **压力测试**：快速连续操作
- **中断测试**：操作中途关闭/断网
- **脏数据测试**：用历史遗留数据测试

## 优先级划分指南

| 优先级 | 定义 | 覆盖范围 |
|--------|------|----------|
| P0 | 关键路径，阻塞性问题 | 核心功能正常流程 |
| P1 | 重要功能，影响用户体验 | 主要功能分支、常见异常 |
| P2 | 一般功能，边界场景 | 边界值、较少使用的功能 |
| P3 | 锦上添花，极端场景 | 探索式测试、极低概率场景 |
