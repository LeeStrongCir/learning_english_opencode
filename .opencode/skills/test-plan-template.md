## 4. 测试策略

### 4.1 功能测试

- **测试方法**: 黑盒测试 + 边界值分析 + 等价类划分
- **用例设计**: 基于需求场景 + 探索式测试
- **执行方式**: 手工测试为主，核心流程可自动化

### 4.2 接口测试

- **工具**: Postman / JMeter / pytest + requests
- **覆盖范围**: 正常场景 + 异常场景 + 边界值 + 幂等性
- **测试数据**: Mock 数据 + 生产脱敏数据

### 4.3 回归测试

- **范围**: 受影响的功能 + 核心功能
- **方式**: 自动化优先 + 手工补充
- **时机**: 每次代码合并后 / 每日构建
