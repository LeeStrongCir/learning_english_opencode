## 1. 项目概述
### 1.1 测试目标
验证阿里云登录页面的功能完整性、安全性、兼容性和性能表现，确保用户能够安全、便捷地完成身份认证并访问阿里云服务。

### 1.2 测试范围
- 账号密码登录功能
- 短信验证码登录功能
- 阿里云App扫码登录功能
- Passkey（无密码）登录功能
- 登录安全机制（验证码、CSRF、设备指纹）
- 多浏览器/多端兼容性
- 登录页面性能表现

### 1.3 不在范围内
- 注册功能（独立测试）
- 第三方登录（微信、GitHub等，如有则单独分析）
- 后端认证服务的单元测试
- 阿里云控制台内部功能

### 2.2 功能详细说明

#### F001 - 账号密码登录
- **功能描述**：用户输入账号（手机号/邮箱/会员名）和密码完成登录
- **输入**：
  - 账号字段：支持手机号（11位）、邮箱（含@）、阿里云会员名
  - 密码字段：6-20位字符，支持字母、数字、特殊字符
  - 图形验证码（触发时）
- **处理逻辑**：
  1. 前端校验输入格式
  2. 发送登录请求至 passport.aliyun.com
  3. 后端验证凭证
  4. 返回登录结果
- **输出**：登录成功跳转 / 登录失败提示
- **业务规则**：
  - 密码输入框默认隐藏，支持显示/隐藏切换
  - 连续密码错误5次触发图形验证码
  - 连续密码错误10次账号锁定30分钟
  - 支持回车键提交登录

#### F002 - 短信验证码登录
- **功能描述**：用户输入手机号，获取短信验证码后完成登录
- **输入**：
  - 手机号：11位中国大陆手机号
  - 短信验证码：6位数字
- **处理逻辑**：
  1. 输入手机号并校验格式
  2. 点击"获取验证码"按钮
  3. 触发人机验证（滑块/图形）
  4. 发送短信验证码
  5. 用户输入验证码并提交
- **输出**：登录成功跳转 / 验证码错误提示
- **业务规则**：
  - 验证码有效期5分钟
  - 获取验证码间隔60秒（倒计时）
  - 同一手机号每日最多获取10次验证码
  - 同一IP每日最多请求50次验证码
  - 验证码输入错误5次后需重新获取

#### F003 - App扫码登录
- **功能描述**：用户使用阿里云App扫描页面二维码完成登录
- **输入**：无（页面展示二维码）
- **处理逻辑**：
  1. 页面加载时生成唯一二维码（含登录凭证）
  2. 用户使用阿里云App扫描二维码
  3. App端确认登录
  4. 页面轮询/长连接接收登录结果
  5. 自动跳转至目标页面
- **输出**：登录成功跳转 / 二维码过期提示
- **业务规则**：
  - 二维码有效期5分钟
  - 过期后自动刷新二维码
  - 支持手动刷新二维码
  - 扫码后App端需确认登录操作
  - 二维码被扫描后状态变更为"已扫描，请在手机上确认"

#### F004 - Passkey登录
- **功能描述**：使用Passkey（FIDO2/WebAuthn）进行无密码登录
- **输入**：生物识别（指纹/面容）或安全密钥
- **处理逻辑**：
  1. 用户选择Passkey登录
  2. 浏览器调用WebAuthn API
  3. 用户完成生物识别或安全密钥验证
  4. 验证凭证发送至服务端
  5. 完成登录
- **输出**：登录成功跳转 / 验证失败提示
- **业务规则**：
  - 需要浏览器支持WebAuthn
  - 需要用户预先绑定Passkey
  - 不支持时展示降级提示

#### F005 - 登录方式切换
- **功能描述**：用户可在密码登录、短信登录、扫码登录之间切换
- **业务规则**：
  - 切换时清空当前输入内容
  - 切换时重置验证码状态
  - 保持页面整体布局稳定

#### F006 - 记住登录状态
- **功能描述**：勾选后下次访问自动填充账号或保持登录
- **业务规则**：
  - 使用持久化Cookie存储
  - Cookie有效期30天
  - 不存储密码明文

#### F007 - 忘记密码
- **功能描述**：跳转至密码找回流程
- **业务规则**：
  - 通过手机号或邮箱验证身份
  - 支持重置密码

#### F009/F010 - 验证码机制
- **功能描述**：防刷机制，包括图形验证码和滑块验证码
- **业务规则**：
  - 连续失败触发
  - 高风险IP直接触发
  - umidToken设备指纹辅助判断

#### F014 - 账号锁定机制
- **功能描述**：连续登录失败后临时锁定账号
- **业务规则**：
  - 同一账号连续失败10次锁定30分钟
  - 同一IP连续失败50次锁定1小时
  - 锁定期间提示剩余时间
  - 支持通过短信验证码解锁

## 3. 非功能需求
### 3.1 性能要求
| 指标 | 要求 |
|------|------|
| 页面首屏加载时间 | ≤ 2秒（4G网络） |
| 登录接口响应时间 | ≤ 1秒（P95） |
| 短信验证码发送延迟 | ≤ 10秒 |
| 二维码生成时间 | ≤ 500ms |
| 扫码登录确认到跳转 | ≤ 2秒 |
| 页面并发承载能力 | ≥ 10000 QPS |

### 3.2 安全要求
| 安全项 | 要求 |
|--------|------|
| 传输加密 | 全站HTTPS，TLS 1.2+ |
| 密码存储 | 后端加盐哈希存储，前端不存储 |
| CSRF防护 | 所有登录请求携带CSRF Token |
| 防暴力破解 | 连续失败锁定、验证码降级 |
| 防重放攻击 | 请求携带时间戳和签名 |
| 设备指纹 | umidToken采集设备信息 |
| XSS防护 | 输入输出转义，CSP策略 |
| SQL注入防护 | 参数化查询 |
| 敏感信息脱敏 | 手机号、邮箱展示时脱敏 |
| Cookie安全 | HttpOnly、Secure、SameSite |

### 4.2 自动化可行性分析
| 功能 | 自动化可行性 | 理由 |
|------|------------|------|
| 账号密码登录 | 高 | 流程固定，输入输出明确 |
| 短信验证码登录 | 中 | 需要短信验证码Mock或测试通道 |
| App扫码登录 | 低 | 需要真实App配合，自动化成本高 |
| Passkey登录 | 低 | 依赖浏览器WebAuthn和生物识别 |

| 验证码触发 | 中 | 触发条件可控，但验证码识别复杂 |
| 账号锁定 | 高 | 可通过脚本模拟多次失败 |
| 页面加载性能 | 高 | 使用Lighthouse等工具 |
| 安全测试 | 中 | 部分可自动化（SQL注入、XSS），渗透测试需人工 |
| 兼容性测试 | 中 | 可使用Selenium Grid/Playwright多浏览器 |

## 5. 风险评估
### 5.1 高风险区域
| 风险项 | 风险描述 | 影响 | 概率 |
|--------|---------|------|------|
| 认证服务依赖 | 登录依赖 passport.aliyun.com，该服务故障将导致全站不可登录 | 高 | 低 |
| 短信服务异常 | 短信通道故障导致验证码无法发送 | 高 | 中 |
| 安全漏洞 | 登录接口存在暴力破解、SQL注入等漏洞 | 高 | 低 |
| 并发峰值 | 大促活动期间登录请求激增导致服务降级 | 高 | 中 |

### 5.2 中风险区域
| 风险项 | 风险描述 | 影响 | 概率 |
|--------|---------|------|------|
| 浏览器兼容 | 新浏览器版本导致页面渲染异常 | 中 | 中 |
| 验证码误判 | 人机验证误拦截正常用户 | 中 | 中 |
| 二维码过期 | 网络延迟导致二维码状态不同步 | 中 | 低 |
| Cookie异常 | 浏览器策略变更导致记住登录失效 | 中 | 低 |

### 5.3 风险缓解建议
1. **认证服务依赖**：建立降级方案，短信登录作为密码登录的备选
2. **短信服务异常**：接入多短信通道，建立通道自动切换机制
3. **安全漏洞**：定期进行安全审计和渗透测试，接入WAF防护
4. **并发峰值**：提前进行压测，建立弹性扩容机制
5. **浏览器兼容**：建立自动化兼容性测试流水线
6. **验证码误判**：优化风控模型，提供人工申诉通道
