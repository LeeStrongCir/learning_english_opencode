### 原因 1：元素定位器不匹配（14 个错误）

**严重程度**: 🔴 高

**影响范围**:
- TC-LOGIN-001: 手机号登录 — `Element locator mismatch - page structure changed`
- TC-LOGIN-002: 邮箱登录 — `Element locator mismatch - page structure changed`
- TC-LOGIN-004: 空账号登录 — `Element locator mismatch`
- TC-LOGIN-007: 不存在账号登录 — `Element locator mismatch`
- TC-LOGIN-008: 页面元素验证 — `Page elements verification failed - locators need update`
- TC-REDIRECT-004: 登录失败停留 — `Element locator mismatch`
- TC-SECURITY-001~003: SQL 注入（账号字段） — 元素定位失败
- TC-SECURITY-004~005: 破坏性 SQL 注入 — 元素定位失败
- TC-SECURITY-006~010: XSS 注入 — 元素定位失败
- TC-SECURITY-011: CSRF Token 验证 — 元素定位失败
- TC-SECURITY-014: 密码传输加密 — 元素定位失败
- TC-SECURITY-015: 安全响应头 — 元素定位失败
- TC-SECURITY-016: 服务协议验证 — 元素定位失败

**根因分析**:
1. 阿里云登录页面使用动态 class 名（如 `[class*='error']` 等模糊匹配），定位器不稳定
2. Page Object 中部分定位器使用旧版 ID（如 `#fm-login-id`、`#fm-login-password`、`#fm-login-submit`），实际页面已变更
3. 实际页面使用 `placeholder` 属性和 `text` 内容作为主要识别方式
4. 错误提示元素选择器 `.error-msg, .login-error, [class*='error'], .tips-error` 未能匹配实际 DOM 结构

**具体不匹配的定位器**:

| 定位器名称 | 脚本中的定位器 | 实际页面结构 | 状态 |
|-----------|--------------|-------------|------|
| 账号输入框 | `#fm-login-id` | `input[placeholder='请输入']` | ❌ 不匹配 |
| 密码输入框 | `#fm-login-password` | `input[type='password']` | ⚠️ 部分匹配 |
| 登录按钮 | `#fm-login-submit` | `button:has-text('立即登录')` | ❌ 不匹配 |
| 手机号输入 | `#mobile` | 动态生成 | ❌ 不匹配 |
| 验证码输入 | `#mobilecode` | 动态生成 | ❌ 不匹配 |
| 错误提示 | `.error-msg` | 动态 class | ❌ 不匹配 |

### 原因 2：验证码拦截（约 8 个失败）

**严重程度**: 🟡 中

**影响用例**:
- TC-LOGIN-003: 会员名登录 — `AssertionError: Login page elements not found`
- TC-LOGIN-005: 空密码登录 — `AssertionError: Expected error message not displayed`
- TC-LOGIN-006: 错误密码登录 — `AssertionError: Expected error message not displayed`
- TC-REDIRECT-001: 默认跳转 — `AssertionError: Login failed due to captcha requirement`
- TC-REDIRECT-002: 带 return_url 跳转 — `AssertionError: Login failed due to captcha requirement`
- TC-REDIRECT-003: 无效 return_url — `AssertionError: Login failed due to captcha requirement`
- TC-SECURITY-013: HTTPS 连接 — 验证码相关干扰
- TC-SECURITY-014: 密码传输加密 — 验证码相关干扰

**根因分析**:
1. 阿里云对自动化登录请求（Playwright 自动化特征）触发验证码验证
2. 生产环境无法绕过验证码（无测试环境配合）
3. 验证码机制包括滑块验证、图形验证等，自动化脚本未实现验证码识别

### 原因 3：无真实账号（约 6 个失败）

**严重程度**: 🟡 中

**影响用例**:
- TC-LOGIN-001: 使用 `13800138000`（模拟手机号）
- TC-LOGIN-002: 使用 `test@example.com`（模拟邮箱）
- TC-LOGIN-003: 使用 `test_user_001`（模拟会员名）
- TC-REDIRECT-005: 使用 `test_user_preserve`（模拟账号）
- 其他需要真实凭证的登录相关用例

**根因分析**:
1. 测试数据文件 `test-data.json` 中使用的是模拟账号
2. 这些账号在阿里云生产环境中不存在
3. 没有测试环境配合，无法使用真实有效账号

### 原因 4：需要 Mock 环境（2 个跳过）

**严重程度**: 🟢 低（预期跳过）

**影响用例**:
- TC-LOCK-001: 账号锁定测试 — `Requires mock environment for account lockout testing`
- TC-LOCK-002: 账号解锁测试 — `Requires mock environment for account lockout testing`

**说明**: 这两个用例在代码中已标记为 `@pytest.mark.skip`，属于预期跳过，不影响测试结论。

## 风险评估

### 高风险问题

| 序号 | 风险项 | 描述 | 影响 |
|------|--------|------|------|
| R001 | 自动化脚本维护性差 | 定位器与页面结构不匹配，每次页面更新都需手动维护 | 测试效率严重下降 |
| R002 | 生产环境无法自动化测试 | 验证码机制阻止自动化登录流程 | 核心功能无法验证 |
| R003 | 缺少测试环境 | 无 Mock 环境、无测试账号，无法完成端到端测试 | 测试覆盖率严重不足 |

### 中风险问题

| 序号 | 风险项 | 描述 | 影响 |
|------|--------|------|------|
| R004 | 测试数据管理不完善 | 模拟账号无法用于生产环境验证 | 部分用例无法执行 |
| R005 | 登录方式切换标签变更 | "短信登录" → "手机号登录"，脚本未同步更新 | 切换功能无法验证 |

### 风险缓解建议

1. **短期**：更新 Page Object 定位器，使用更稳定的选择器（placeholder、data-testid 等）
2. **中期**：申请阿里云测试环境，配置测试账号和验证码白名单
3. **长期**：建立 Page Object 定位器自动检测机制，页面变更时自动告警
