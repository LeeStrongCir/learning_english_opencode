## 缺陷详情

### TC-001: 多处 CSS 类名断言使用精确匹配

**基本信息**:
- 缺陷类型：测试用例缺陷
- 严重程度：中
- 功能模块：年级选择、册次选择、单元选择、面包屑导航
- 发现日期：2026-04-18
- 状态：Open
- 影响用例数：10

**问题描述**:
多个测试用例在使用 Playwright 的 `to_have_class()` 断言时，使用了精确匹配模式。当目标元素包含多个 CSS 类（如 `class="nav-btn active"`）时，断言期望值仅为 `"active"`，导致匹配失败。

**复现步骤**:
1. 运行涉及按钮激活状态断言的测试用例
2. 测试执行到 `expect(element).to_have_class("active")` 断言
3. 断言失败，报错信息：期望 `"active"`，实际得到 `"nav-btn active"` 等

**预期行为**: 
测试用例应正确识别元素包含 `active` 类，无论是否还有其他类

**实际行为**: 
断言失败，因为 `to_have_class("active")` 默认进行精确匹配

**影响范围**:
- 不影响产品功能
- 影响测试通过率统计（10 个用例报告为失败）
- 可能导致误判产品存在缺陷

**涉及测试用例**:
| 序号 | 测试模块 | 涉及元素 | 实际 class 值 |
|------|---------|---------|-------------|
| 1 | 年级选择 | 年级按钮 | `nav-btn active` |
| 2 | 年级选择 | 年级按钮 | `nav-btn active` |
| 3 | 册次选择 | 册次按钮 | `volume-btn active` |
| 4 | 册次选择 | 册次按钮 | `volume-btn active` |
| 5 | 单元选择 | 单元按钮 | `unit-btn active` |
| 6 | 单元选择 | 单元按钮 | `unit-btn active` |
| 7 | 面包屑导航 | 导航按钮 | `nav-btn active` |
| 8 | 面包屑导航 | 导航按钮 | `nav-btn active` |
| 9 | 其他 | 其他按钮 | `*-btn active` |
| 10 | 其他 | 其他按钮 | `*-btn active` |

**修复建议**:

#### 方案一：使用正则表达式匹配（推荐）
```python
# 修改前
expect(element).to_have_class("active")

# 修改后
expect(element).to_have_class(re.compile(r".*\bactive\b.*"))
```

#### 方案二：使用部分匹配模式
```python
# 修改前
expect(element).to_have_class("active")

# 修改后
expect(element).to_have_class("active", exact=False)
```

#### 方案三：检查 class 属性包含
```python
# 修改前
expect(element).to_have_class("active")

# 修改后
classes = element.get_attribute("class")
assert "active" in classes.split()
```

**优先级**: 中
**建议修复时间**: 下次测试执行前
